Creating Your First Application

You can create an application either from within the network page or from the Application listing page. Creating a new network on the platform is fairly simple. You'll need these information beforehand:â€‹

  1. A name and an optional description for the application
  2. Technical details including:
    • IPv4 address.
    • Port number.
    • Network protocol TCP, UDP or ICMP.
    • DNS name (if any).
  3. An icon from a selection of icons.
    note

    The selected icon will be shown for the user when connecting using FrontierZero client agents.

  4. FrontierZero special attributes:
    • AlwaysOn: When an application has this attribute enabled, the application becomes always available to all users connecting to the network. Access policies will not apply to it.
      tip

      Use the AlwaysOn attribute for applications that all users should have access to. Some examples:

      - Microsoft Active Directory Domain Controller ports.
      - Internal Email servers.
      - Knowledgebases and Wikis that should be accessible by everyone.
    • Agentless Protocol: When this is set, the application will be accessible from both agent-based clients and from within the browser. See Agentless Applications for more information.
      caution

      Only use the Agentless attribute for applications with supported protocols. Current supported protocols are RDP, SSH and VNC.

    • WebApp: When this attribute is set and an HTTP protocol is selected, the application will be treated as a webserver. This is particularly helpful in virtual hosting and SSL interception. See Web Applications for more information.
